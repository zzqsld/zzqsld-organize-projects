# CONTRIBUTING.md

# 贡献指南

感谢您对 `organize-projects` 项目的关注！我们欢迎任何形式的贡献，包括报告问题、提交功能请求、修复错误和改进文档。请遵循以下指南以帮助我们更好地管理贡献。

## 如何贡献

1. **报告问题**
   - 如果您发现了错误或有功能请求，请在 GitHub 的 [问题](https://github.com/yourusername/organize-projects/issues) 页面上提交一个新问题。请尽量提供详细的信息，包括重现步骤和预期结果。

2. **提交代码**
   - 请遵循以下步骤提交代码：
     1. **Fork** 本仓库。
     2. 在您的本地机器上克隆您的 fork：
        ```
        git clone https://github.com/yourusername/organize-projects.git
        ```
     3. 创建一个新的分支：
        ```
        git checkout -b feature/your-feature-name
        ```
     4. 在您的分支上进行更改并提交：
        ```
        git commit -m "描述您的更改"
        ```
     5. 将更改推送到您的 fork：
        ```
        git push origin feature/your-feature-name
        ```
     6. 在 GitHub 上创建一个新的 Pull Request，描述您的更改和其目的。

3. **代码风格**
   - 请确保您的代码遵循项目的代码风格。我们使用 [PEP 8](https://www.python.org/dev/peps/pep-0008/) 作为 Python 代码的风格指南。

4. **测试**
   - 在提交代码之前，请确保所有测试都通过。您可以使用以下命令运行测试：
     ```
     pytest
     ```

5. **文档**
   - 如果您添加了新功能或更改了现有功能，请确保更新相关文档。

## 其他信息

- 请查看 [README.md](README.md) 以获取有关项目的更多信息。
- 如果您有任何问题或需要帮助，请随时在 GitHub 上联系项目维护者。

感谢您的贡献！我们期待您的参与！
